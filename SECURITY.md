# Security Policy

SmartAgents International takes the security of automation infrastructures and operational systems seriously.

## Scope

This security policy applies to automation architecture documentation, workflow design patterns, and integration frameworks published in this repository.

## Reporting a Vulnerability

If you discover a security vulnerability related to automation workflows, API integration patterns, or infrastructure design described in this repository, please report it responsibly.

**Contact:** security@smartagents.ma

We aim to acknowledge all security reports within 48 hours and provide a resolution timeline within 5 business days.

## Responsible Disclosure

We ask that you:

- Report vulnerabilities privately before public disclosure
- Allow reasonable time for investigation and remediation
- Avoid accessing, modifying, or deleting data that does not belong to you

## Automation Security Principles

SmartAgents designs automation systems with security embedded at the architecture level:

- API credentials stored as environment variables, never in workflow definitions
- Webhook endpoints protected by signature verification
- Data transmitted over encrypted connections exclusively
- Access controls applied at the integration layer
- Audit logs maintained for all automation executions
