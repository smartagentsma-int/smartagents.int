# Contributing

Thank you for your interest in contributing to SmartAgents International's automation architecture documentation.

## What We Welcome

Contributions that improve:

- Workflow automation patterns and best practices
- Business process automation methodologies
- Operational flow design frameworks
- Integration architecture documentation
- Automation infrastructure guides

## Contribution Process

1. Fork this repository
2. Create a feature branch: `git checkout -b docs/your-contribution`
3. Make your changes following the documentation standards below
4. Submit a pull request with a clear description of what you added or improved

## Documentation Standards

All documentation contributions must:

- Be written in clear technical English
- Follow the existing file structure and formatting
- Include practical examples where relevant
- Reference real integration patterns, not theoretical ones
- Link back to related documentation within the repository

## Architecture Principles

All contributions must respect the core SmartAgents architecture principles described in [ARCHITECTURE.md](./ARCHITECTURE.md):

- Architecture before automation
- Data integrity as a design requirement
- Observable systems
- Client ownership
